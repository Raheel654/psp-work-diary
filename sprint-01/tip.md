During this sprint week, the team started properly working on the project. We had a short call every morning to plan tasks and another at the end of the day to share updates and discuss any issues. This helped keep everyone on track.
My main role was working on the backend and helping connect different parts of the system. At the beginning, there were some problems setting up the backend environment in Visual Studio 2022, so I spent some time fixing configuration issues and getting everything running.
We also faced a few technical challenges as a team. The frontend was mainly developed in VS Code while the backend used Visual Studio 2022. Some team members were using Mac devices, which made it difficult for them to run Visual Studio, causing some delays.
There were also a few coding bugs while connecting the backend components, which took time to debug. However, the daily calls helped us quickly discuss problems and find solutions together.
By the end of the sprint week, we had made good progress with the backend setup and had a clearer workflow for the next stage of development.
